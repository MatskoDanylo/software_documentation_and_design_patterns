# Laboratory Work 1: UML Modeling

**Variant:** 17 
**Domain:** SAP SuccessFactors (people onboarding)

## Assignment Overview
This laboratory work consists of four distinct parts, each requiring the implementation of a specific UML diagram.

## Tasks

### 1. Use Case Diagram
Draw a use-case diagram illustrating the onboarding process of a new employee in an IT company. The diagram must clearly identify and represent the primary actors.

### 2. Class Diagram
Create a class diagram representing the set of objects required to facilitate the new employee's onboarding process and ensure they are provided with everything necessary for their work. The design must utilize OOP principles: classes, inheritance, polymorphism, and encapsulation. The diagram should only depict the static structure relevant to this specific task.

### 3. Activity Diagram
Draw an activity diagram detailing the new employee's first day at the IT company, covering the formal registration and instruction procedures. Use timers on the diagram only if necessary.

### 4. Sequence Diagram
Create a sequence diagram demonstrating the interaction of SAP SuccessFactors with reporting systems, compensation calculation, leave accrual, and LMS systems.
